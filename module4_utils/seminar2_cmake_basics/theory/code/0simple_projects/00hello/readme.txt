Это простейший проект с использованием CMake.

Для сборки этого проекта создайте новую папку, например с названием bin, 
затем в терминале зайдите в эту папку и исполните:

    cmake ..
    cmake --build .

После этого в папке bin появится исполняемый файл.


Сборку проекта в CMake можно разделить на 2 шага: конфигурация и компиляция.

1) Шаг конфигурации

    Первая команда:

        cmake ..

    Это шаг конфигурации проекта. На этом шаге CMake нужно передать путь до основной папки проекта.
    В этой папке должен находиться главный CMake файл проекта под названием CMakeLists.txt.
    В данном примере мы находимся в папке bin, а файл CMakeLists.txt лежит на одну директорию выше,
    поэтому путь до основной папки будет просто состоять из двух точек.
    (две точки в терминале обозначают папку, которая находится на один уровень выше той, в которой мы находимся)

    На этапе конфигурации CMake находит файл CMakeLists.txt и выполняет его.
    После этого в текущей папке создадутся все необходимые файлы для последующей компиляции.

    Файл CMakeLists.txt - это основной файл проекта. Он содержит код написанный 
    на языке CMake и описывает как нужно собирать тот или иной проект.
    Данный проект очень простой, поэтому код содержит всего 3 строки:

        cmake_minimum_required(VERSION 3.20)
        project(helloProject)
        add_executable(hello hello.cpp)

    cmake_minimum_required  -   задаём минимальную возможную версию CMake. попытка компиляции 
                                с помощью CMake-а меньшей версии приведёт к ошибке.
    project                 -   устанавливаем имя проекта
    add_executable          -   задаём исполняемый файл (hello) и файлы из которых он будет 
                                собираться (в данном случае это всего один файл hello.cpp)


2) Шаг компиляции

    Вторая команда:

        cmake --build .

    Это шаг компиляции проекта. Мы передаём CMake опцию --build, чтобы указать что это шаг
    компиляции. После этого передаём CMake путь до папки, в которой происходил шаг конфигурации.
    В данном случае эта текущая папка, поэтому путь будет состоять просто из одной точки.
    (одна точка в терминале обозначают папку, в которой мы сейчас находимся)


После шага компиляции в папке появится итоговый исполняемый файл под названием hello (если
проект собирался на Linux) или hello.exe (если проект собирался на Windows).
Программу можно запустить следующей командой:

    ./hello    (на Linux)          .\hello.exe     (на Windows)
